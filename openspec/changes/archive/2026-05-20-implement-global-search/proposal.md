## Why

右上角搜索目前只是一个无行为图标，但 Fox Procureflow 已经有采购申请、审批、RFQ、采购订单、收货发票、三单匹配和供应商池等多个工作台。演示人员和采购用户需要通过单据号、供应商、标题、发票号或人员线索快速定位业务对象，而不是逐页切换查找。

## What Changes

- 将右上角搜索入口升级为全局采购搜索弹窗，支持点击入口和键盘快捷键打开。
- 新增后端只读搜索 API，聚合当前 MVP 采购闭环中的业务对象：采购申请、审批、RFQ、采购订单、收货/发票、三单匹配、供应商池和核心主数据。
- 搜索结果按业务类型分组，展示编号、标题、状态、公司、供应商、金额、时间和命中字段等足够上下文。
- 点击搜索结果后跳转到对应工作台，并尽量打开对应详情抽屉或定位到相关记录。
- 搜索遵守现有多公司边界：交易数据默认按当前公司检索，集团共享供应商池和采购品类可跨公司共享展示。
- 增加加载、空态、错误态和键盘导航体验，避免搜索弹窗成为静态装饰。
- 不改变采购申请、审批、RFQ、采购订单、收货、发票或三单匹配的状态流转。
- 不引入 Elasticsearch、Redis、AI 语义搜索、外部搜索服务或新的权限系统。

## Capabilities

### New Capabilities

- `global-search`: 定义采购工作台全局搜索能力，包括搜索入口、后端只读聚合 API、结果分组、跳转行为、多公司数据边界和前端弹窗体验。

### Modified Capabilities

- None.

## Impact

- 后端新增全局搜索 controller/service/dto，复用现有 MySQL 数据和 master data 查询，不新增业务状态表。
- 前端 topbar 搜索按钮从占位按钮变为全局搜索弹窗入口，并新增搜索结果导航到现有工作台。
- 可能需要扩展现有详情打开方式，例如通过 URL query 或内部状态在跳转后选中目标记录。
- OpenAPI 文档新增搜索端点说明。
- 测试覆盖后端公司隔离、集团共享数据、搜索结果排序与前端弹窗行为。

## Non-Goals

- 不实现跨集团、多租户、外部 SaaS 搜索或用户自定义搜索范围。
- 不做全文搜索引擎、拼音分词、模糊语义召回或 AI 问答。
- 不从搜索结果直接创建、审批、发布、收货、开票、处理三单匹配或上传附件。
- 不改变现有 sidebar 数字、看板指标或各业务列表自身的筛选规则。

## Acceptance Criteria

- 用户点击右上角搜索或按快捷键后，可以打开全局搜索弹窗并输入关键词。
- 输入 `PR`、`RFQ`、`PO`、供应商名称、发票号或三单匹配相关线索时，系统返回真实后端搜索结果，而不是前端静态 mock。
- 当前公司为星河数字科技有限公司时，交易类结果不得包含星河智能制造有限公司的公司级采购记录；集团共享供应商池结果仍可展示。
- 搜索结果可跳转到对应工作台，并打开或定位到目标记录详情。
- 搜索 API 和弹窗均为只读行为，不改变任何采购业务状态。
