## Why

当前系统已经具备集团共享供应商池主数据和只读 API，但前端侧边栏中的“供应商池”只是静态导航项，真实供应商表格藏在“组织与主数据”页面中，演示时容易被理解为供应商池页面不可用。

在进入 roadmap 后续“真实附件上传”之前，应先把集团共享供应商池补成一个一眼可用的独立工作台入口，方便采购员在 RFQ、PO 和看板之外查看供应商覆盖品类、风险等级和集团共享边界。

## What Changes

- 新增“供应商池”前端工作台页面，路由建议为 `/suppliers`，并将侧边栏“供应商池”变成可点击导航入口。
- 页面从现有 `/api/master-data/suppliers`、`/api/master-data/categories` 和上下文 API 读取真实后端数据，不使用前端静态 mock 数据。
- 支持按品类、风险等级、状态和关键词筛选集团共享供应商，并展示至少 5 家 roadmap seed 供应商。
- 支持供应商详情抽屉，展示服务范围、所在地、风险等级、状态、共享范围、覆盖品类，以及其作为集团共享参考数据与公司级采购交易数据的边界说明。
- 保持供应商池只读，不提供创建、编辑、删除、审批、RFQ、PO、收货、发票或三单匹配操作。
- 非目标：不实现供应商主数据维护、供应商门户、供应商准入流程、资质附件、绩效评分计算、外部账号体系、真实文件上传或 MinIO 存储。
- 验收标准：启动本地前后端后，用户可以从侧边栏打开供应商池页面，看到来自后端的集团共享供应商列表，使用筛选和详情抽屉确认两家公司共享同一供应商池，同时不出现任何会改变供应商或采购交易状态的操作。

## Capabilities

### New Capabilities

- `supplier-pool`: 定义集团共享供应商池的独立前端工作台体验，包括导航入口、只读列表、筛选、详情抽屉和共享/隔离边界展示。

### Modified Capabilities

- None.

## Impact

- 影响前端：React 路由、侧边栏导航、供应商池页面组件、筛选状态、详情抽屉、响应式样式和现有 master data query 复用。
- 影响 API：优先复用现有只读 `/api/master-data/**` API；如发现前端筛选需要服务端支持，可仅扩展只读查询参数，不新增供应商写接口。
- 影响数据：不新增业务表，不改变既有 Flyway seed 数据；供应商仍是集团共享参考数据，采购申请、RFQ、PO、收货、发票和三单匹配仍按公司隔离。
- 影响测试：补充前端构建/页面验证，并保留或补充后端 master data API 测试以确认供应商池仍返回 5 家演示供应商及品类关联。
