## Why

当前系统已经完成采购申请 Intake 和审批流，但审批通过后的采购执行链路仍然断在采购员发起询价之前。RFQ 询报价切片把“已审批采购申请”转化为可比较、可推荐、可进入后续 PO 的采购执行记录，是 Fox Procureflow MVP 从内部申请审批走向供应商协同和三单匹配闭环的下一步。

## What Changes

- 新增 RFQ 询报价业务垂直切片，采购员只能基于已审批通过的采购申请创建 RFQ。
- 支持从采购申请带入公司、申请人、品类、预算科目、金额、交付日期和明细行快照，并保持公司维度数据隔离。
- 支持采购员从集团共享供应商池选择候选供应商，发起 RFQ，并录入每家供应商的报价、交期、评分、风险备注和附件元数据占位。
- 支持 RFQ 列表、详情、报价对比和推荐排序，帮助采购员选择后续 PO 的候选中标报价。
- 前端新增“询报价”工作区视图，覆盖从已审批申请创建 RFQ、供应商选择、报价录入、报价对比和查看推荐结果的演示流程。
- 非目标：本 change 不生成采购订单，不实现供应商外部门户，不引入真实附件上传、MinIO、RabbitMQ、Redis 指标缓存、AI 报价解释或自动中标决策。

## Capabilities

### New Capabilities

- `rfq`: 覆盖基于已审批采购申请创建 RFQ、选择供应商、录入报价、维护附件元数据占位、报价对比、推荐排序、公司隔离 API 和前端询报价工作流。

### Modified Capabilities

本 change 不修改现有 capability 的 requirement。采购申请和审批流保持现有职责，RFQ capability 通过读取已审批申请和审批结果来建立下游采购执行记录。

## Impact

- 后端新增 RFQ 领域模型、Flyway 表结构、Repository、Service、Controller、DTO、校验和 Swagger 文档。
- 后端读取采购申请、审批结果、集团共享供应商、公司、用户、品类、预算科目等现有主数据和业务记录。
- 前端新增 RFQ 页面、路由、API client、TanStack Query hooks、创建 RFQ 表单、报价录入表单、报价对比表和推荐展示。
- 测试新增 RFQ 创建资格校验、公司隔离、供应商选择、报价录入、报价对比排序、API 文档和前端核心流程覆盖。
- 验收标准：演示人员可以使用星河数字科技已审批 IT 设备采购申请创建 RFQ，选择三家供应商，录入报价和交期，在报价对比中看到价格、交期、评分、风险和推荐排序；未审批或跨公司采购申请不能创建 RFQ。
